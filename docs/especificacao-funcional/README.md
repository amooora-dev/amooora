# Especificações funcionais – Amooora

Esta pasta reúne as especificações funcionais por área do app, para apoio a desenvolvedores e manutenção.

## Manutenção dos documentos

**Sempre que uma página ou fluxo for alterado no código, atualize o documento correspondente nesta pasta.** Inclua as mudanças nas seções afetadas (rotas, blocos, ações, arquivos) e **atualize a data "Última atualização" no início do documento** para a data da alteração. Assim a equipe mantém as specs alinhadas ao que está implementado.

| Documento | Área |
|-----------|------|
| [SPEC_EVENTOS.md](./SPEC_EVENTOS.md) | Eventos |
| [SPEC_HOME.md](./SPEC_HOME.md) | Home |
| [SPEC_LOCAIS.md](./SPEC_LOCAIS.md) | Locais (Lugares Seguros) |
| [SPEC_COMUNIDADE.md](./SPEC_COMUNIDADE.md) | Comunidade |
| [SPEC_PERFIL.md](./SPEC_PERFIL.md) | Página de Perfil |
| [SPEC_SERVICOS.md](./SPEC_SERVICOS.md) | Serviços |
| [SPEC_MAPA.md](./SPEC_MAPA.md) | Mapa |
| [SPEC_SOBRE_AMOORA.md](./SPEC_SOBRE_AMOORA.md) | Sobre Amooora |
| [SPEC_FALE_CONOSCO.md](./SPEC_FALE_CONOSCO.md) | Fale Conosco |
| [SPEC_LOGIN.md](./SPEC_LOGIN.md) | Login |

Cada MD descreve rotas, fontes de dados, blocos da tela, ações do usuário e arquivos principais. No início de cada documento há o campo **Última atualização** (data); revise esse campo sempre que alterar o conteúdo da spec.
